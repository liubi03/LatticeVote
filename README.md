# LatticeVote: A Post-Quantum Voting Demo

基于格密码的同态加密电子投票演示系统

## 项目介绍

本项目是**量子密码与后量子密码课程中期展示**作品，基于论文《A Homomorphic LWE Based E-voting Scheme》实现了一个基于 LWE（Learning With Errors）的同态加密投票系统。

### 项目背景

随着量子计算的发展，传统的基于数论困难问题（如 RSA、ECC）的加密方案面临被量子算法破解的风险。基于格密码的后量子密码学被认为是抵御量子计算攻击的最有前途的方向之一。本项目演示了如何利用 BFV 同态加密方案构建一个隐私保护的电子投票系统。

### 核心技术

- **BFV 同态加密方案**：基于 LWE 问题，支持在密文上直接进行加法运算
- **同态计票**：选票加密后直接累加，无需解密即可统计票数
- **隐私保护**：整个计票过程不暴露任何单张选票的内容

## 核心原理简介

### 同态加密概念

同态加密是一种特殊的加密形式，允许对密文进行特定的代数运算，解密后的结果与对明文进行相同运算的结果一致。

```
Enc(m1) ⊕ Enc(m2) = Enc(m1 ⊕ m2)
```

本项目使用的是**加法同态**：可以在密文上执行加法操作。

### BFV 方案简介

BFV（Brakerski-Fan-Vercauteren）是一种基于格密码的全同态加密方案，本项目使用其加法同态特性：

| 参数 | 说明 | 本项目设置 |
|------|------|-----------|
| poly_modulus_degree | 多项式模次数 | 8192 |
| plain_modulus | 明文模数 | 1032193 |

### 投票流程说明

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   初始化    │ ──▶ │    注册     │ ──▶ │    投票     │ ──▶ │    计票     │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
      │                   │                   │                   │
      ▼                   ▼                   ▼                   ▼
  生成密钥对         选民身份登记        加密投票选择        同态累加选票
  创建公告板         防止重复注册        提交加密选票        解密公布结果
```

#### 1. 初始化阶段
- 生成 BFV 加密上下文（公钥、私钥）
- 创建选民注册表和公告板
- 设置候选人信息

#### 2. 注册阶段
- 选民提交身份信息
- 系统验证身份并登记
- 防止重复注册

#### 3. 投票阶段
- 选民选择候选人
- 系统将选择编码为 one-hot 向量
- 使用公钥加密选票
- 提交加密选票到公告板

#### 4. 计票阶段
- 对所有加密选票进行同态加法
- 一次性解密累加结果
- 公布各候选人得票数

### 安全性优势

| 特性 | 说明 |
|------|------|
| **隐私保护** | 选票以密文形式存储，任何人无法查看单张选票内容 |
| **防篡改** | 同态加密保证计票过程不可被操纵 |
| **防重复投票** | 选民注册表确保一人一票 |
| **可验证性** | 计票结果可通过总票数验证 |

## 技术特点

### 1. 同态加密保护投票隐私

```python
# 选票加密示例
choice_vector = [0, 1, 0, 0]  # 投票给第2位候选人
encrypted_vote = bfv.encrypt_vector(choice_vector)

# 同态计票：密文直接相加
encrypted_sum = encrypted_vote1 + encrypted_vote2 + ...

# 最终解密：只解密累加结果
vote_counts = bfv.decrypt(encrypted_sum)  # [2, 5, 1, 3]
```

### 2. 防止重复投票

- 选民注册表记录所有已注册选民
- 公告板追踪已投票选民 ID
- 提交选票时自动验证投票资格

### 3. 可验证的计票结果

- 总票数与选民数一致性验证
- 各候选人得票数非负性检查
- 计票结果完整性验证

### 4. 交互式 Web 界面

- 直观的投票流程引导
- 实时投票进度显示
- 可视化计票结果（柱状图）
- 响应式界面设计
