# AI生成PPT提示词

## 使用说明
将以下提示词复制到AI工具（如Gamma、Beautiful.ai、Tome等）中生成PPT，然后根据需要进行调整和添加图片。

---

## 提示词

```
请帮我生成一个关于"基于格密码的后量子安全电子投票系统"的学术演示PPT，共10页，用于课程中期展示。

## 整体风格要求
- 学术风格，简洁专业
- 配色：深蓝色为主色调，搭配白色和浅灰色
- 可加入科技感元素：格点阵图案、加密符号、盒子示意图等
- 字体：标题使用粗体，正文使用常规字体
- 重点：通俗易懂，让初学者也能理解协议原理

## PPT结构

### 第1页：封面（30秒）
- 标题：LatticeVote - 基于格密码的后量子安全电子投票系统
- 副标题：量子密码与后量子密码课程中期展示
- 演示人、日期信息
- 背景：科技感，可加入格点阵图案

### 第2页：电子投票的安全挑战（1分钟）
- 标题：电子投票面临的安全挑战
- 三个核心问题：
  1. 隐私性：没人知道我投给了谁
     - 传统方案：信任计票机构
     - 问题：计票机构可能作恶
  2. 正确性：计票结果必须准确
     - 如何证明计票没有作弊？
  3. 可验证性：选民能确认自己的票被计入
     - 但不能证明自己投给了谁（防止买卖选票）
- 加入传统投票 vs 电子投票对比图

### 第3页：核心思想 - 同态加密（2分钟）
- 标题：核心思想：同态加密
- 内容：
  1. 什么是同态加密？
     - 普通加密：加密后不能计算
     - 同态加密：加密后仍可计算，结果正确
  2. 加法同态的直观理解
     - 想象一个"魔法盒子"
     - 把数字放进去，盒子会锁住
     - 可以把两个盒子"合并"，得到一个新盒子
     - 打开新盒子，里面是两个数字的和
  3. 数学表达
     - Enc(a) + Enc(b) = Enc(a + b)
- 加入魔法盒子示意图，展示同态加法过程

### 第4页：选票编码方案（1分30秒）
- 标题：选票编码方案
- 内容：
  1. 问题：如何用数字表示投票选择？
  2. One-Hot编码
     - 假设有3位候选人：Alice, Bob, Carol
     - 投给Alice：[1, 0, 0]
     - 投给Bob：[0, 1, 0]
     - 投给Carol：[0, 0, 1]
  3. 为什么这样编码？
     - 同态加法直接得到票数
     - [1,0,0] + [0,1,0] + [0,1,0] = [1, 2, 0]
     - 结果：Alice 1票，Bob 2票，Carol 0票
  4. 加密后
     - Enc([1,0,0]) + Enc([0,1,0]) + Enc([0,1,0]) = Enc([1,2,0])
     - 计票过程不暴露任何单张选票
- 加入One-Hot编码示意图 + 同态加法示意图

### 第5页：完整投票协议流程（2分钟）
- 标题：完整投票协议流程
- 内容：
  1. 系统初始化
     - 生成加密密钥对（公钥、私钥）
     - 公钥公开，用于加密选票
     - 私钥由多个受托人分片保管
  2. 选民投票
     - 选择候选人 → One-Hot编码
     - 用公钥加密选票
     - 用自己的私钥签名（证明身份）
     - 生成零知识证明（证明选票有效）
  3. 同态计票
     - 验证所有签名和证明
     - 同态累加所有加密选票
  4. 结果解密
     - 受托人联合解密
     - 公布各候选人得票数
- 加入完整的协议流程图，用箭头标注各步骤

### 第6页：安全性分析 - 隐私保护（1分钟）
- 标题：安全性分析：隐私如何保护？
- 内容：
  1. 单张选票的隐私
     - 选票始终以密文形式存在
     - 只有公钥加密，没有私钥无法解密
     - 私钥分片保管，单人无法解密
  2. 计票过程的隐私
     - 同态加法在密文上进行
     - 不需要解密任何单张选票
     - 只解密最终总和
  3. 即使攻击者获取了所有数据
     - 加密选票：无法解密
     - 签名：只能验证，无法伪造
     - 零知识证明：不泄露任何信息

### 第7页：安全性分析 - 正确性与可验证性（1分钟）
- 标题：安全性分析：正确性与可验证性
- 内容：
  1. 正确性保证
     - 数字签名：防止伪造选票
     - 零知识证明：证明选票格式正确
     - 同态运算的数学正确性
  2. 可验证性
     - 选民可以验证自己的签名被记录
     - 任何人可以验证计票过程
     - 结果可审计、可重现
  3. 防作弊机制
     - 防止重复投票（签名唯一性）
     - 防止篡改选票（签名验证）
     - 防止恶意计票（公开可验证）

### 第8页：后量子安全性（1分钟）
- 标题：为什么是"后量子"安全的？
- 内容：
  1. 传统密码学的量子威胁
     - Shor算法：可破解RSA、ECC
     - 量子计算机发展迅速
  2. 格密码的优势
     - 基于LWE问题（带误差学习）
     - 目前没有已知量子算法能有效求解
     - NIST后量子密码标准候选
  3. 我们的选择
     - BFV方案：基于格的同态加密
     - 安全性归约到格问题
     - 即使量子计算机出现也安全

### 第9页：系统实现简介（30秒）
- 标题：系统实现简介
- 内容：
  - 技术栈：Python + TenSEAL（同态加密库）
  - 功能：账户管理、投票、计票、演示
  - Web界面：Streamlit
- 可加入系统界面截图

### 第10页：总结与致谢（30秒）
- 标题：总结
- 核心贡献：
  - 基于格密码的同态加密投票协议
  - 隐私保护：选票全程加密
  - 正确性保证：签名+零知识证明
  - 后量子安全：抵抗量子计算攻击
- 后续工作：
  - 完善零知识证明
  - 分布式密钥管理
  - 性能优化
- 致谢：感谢聆听，欢迎提问

```
